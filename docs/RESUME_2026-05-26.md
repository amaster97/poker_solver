### 1. Gate 4 200K v2 (river + turn) â€” INVALID

  exploitability figures are uniform-random fallback, NOT a converged
  Nash equilibrium.
- **Turn phase (v2):** Killed mid-run by orchestrator at ~60 min
  wall-clock. `/tmp/gate4_200k_v2_turn.log` is 0 bytes.
- **Root cause:** `solve --hunl-mode postflop --backend rust`
  invocation WITHOUT `--initial-hole-cards`. The CLI constructs an
  `HUNLConfig` with `initial_hole_cards = None`, which causes
  `HUNLState::chance_outcomes` (in
  `crates/cfr_core/src/hunl.rs:533-568`) to return `Vec::new()`
  defensively. The scalar CFR loop's chance branch then iterates over
  the empty list, returns `[0.0, 0.0]` immediately, and never inserts
  a single infoset into `strategy_sum`. The 200,000 iters of
  "return [0,0]" complete in ~111 sec wall-clock. Reported
  exploitability is then recomputed by `_rust.compute_exploitability`
  against the empty strategy map and falls back to `uniform(n_actions)`
  on every cache miss (`exploit.rs:472-475`).
- **Note on the 2026-05-25 Gate 4 v1 result (5.28e-14 mbb/g
  river exploitability, monotone clean):** that earlier run used the
  same CLI shape and may have hit the same no-op path. **Treat as
  PROVISIONAL until re-run via the correct entrypoint.**
